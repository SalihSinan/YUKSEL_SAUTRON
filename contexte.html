Contexte du projet

Cette page est dédiée à l'explication du contexte du projet.

Description du projet

Pour ce projet, nous avions réfléchi aux problèmes des programmeurs, et nous avions fait le choix d'aider les programmeurs Python. Nous avons pensé à nos débuts, et nous avons réfléchi à nos problèmes. On s'est dit que les programmeurs Python possèdent beaucoup de tâches longues qui pourraient peut-être être faites à l'aide d'une application. C'est donc pour cela que nous nous sommes mis au travail et avons programmé une application Java pour ne plus avoir à faire toutes ces tâches. Le projet consiste à mettre en œuvre les compétences acquises en Programmation Orientée Objet (POO) en se focalisant sur l'amélioration de la qualité des codes Python. Cela inclut la vérification des annotations de type des variables, l'ajout de commentaires pydoc aux fonctions, ainsi que la possibilité d’ajouter le SHEBANG et l’UFT-8 lorsque ceci est nécessaire. En outre, l'objectif est de développer une Application d'Aide pour les Développeurs Python, avec une interface en ligne de commande (CLI) regroupant toutes les fonctionnalités, et une interface utilisateur graphique (GUI) intégrant des boutons dédiés. La gestion du projet prévoit une collaboration en binôme, de la coopération, et l'établissement d'un système de partage de charge de travail. En résumé, le projet vise à créer une application d'assistance pour les développeurs Python.

Objectifs du projet

Lors de ce projet, nous avons décidé d'aider les programmeurs Python et de diminuer leur charge de travail. Nous avions donc fait le choix de créer deux interfaces : une interface en ligne de commande et une interface graphique. Ce projet permet aux utilisateurs Python d'analyser leurs répertoires, leurs fichiers et de retrouver les fichiers Python d'un répertoire. L'utilisateur pourra générer des statistiques pour un fichier, telles que le pourcentage de fonctions avec pydoc, le pourcentage de fonctions avec un commentaire shebang, le pourcentage de fonctions annotées, le nombre de fichiers analysés et le nombre total de fonctions. Ensuite, il pourra vérifier si un fichier possède des annotations de type, des commentaires pydoc, et les ajouter si nécessaire. Il est également possible de vérifier les annotations. Nous pensions avoir abouti le projet, mais nous envisageons des améliorations pour le futur, telles que l'ajout d'un historique de statistiques et l'édition des fichiers dans le CLI. Nous sommes également ouverts à vos critiques dans notre rubrique Contact.
