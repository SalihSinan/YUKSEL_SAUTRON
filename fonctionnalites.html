Fonctionnalités du projet

Cette partie est dédiée à l'explication des fonctionnalités du projet.
Ce projet possède deux interfaces :

CLI

Cette partie est dédiée à la description de l'interface de ligne de commande.

Description du CLI

La classe CLI représente une interface en ligne de commande qui permet d'effectuer plusieurs actions, pour faciliter et diminuer la tâche des développeurs pythons, sur les axes de l'analyse et l'édition de fichiers pythons. L'utilisateur devra donc seulement taper la commande de la tâche qu'il a envie sur son terminal pour que celle-ci fasse automatiquement l'analyse, ou la modification qu'il veut sur ses fichiers ou répertoire. La méthode help(), permet de lui dire tous les options possibles en expliquant quel commandes utiliser pour lister les fichier pythons d'un répertoire, pour générer les statistiques d'un répertoire, pour analyser les fonctions annoté d'un fichier python, pour analyser le Shebang et UTF8 d'un fichiers, pour analyser les commentaire pydoc d'un fichiers, pour ajouter le Shebang et l'UTF8 ou les commentaires pydoc (si nécessaire). La classe utilise d'autres classes telles que Stat, FichiersPython, Ligne, AnalysePydoc, et AjouterSquelettePydoc pour effectuer des actions différentes selon la nécessité. Cette classe CLI permet donc l'analyse et la gestion de fichiers Python, directement sur le terminal

GUI

Cette partie est dédiée à la description de l'interface graphique.

Description du GUI

Le GUI permet d'avoir une interface conviviale et intuitive permettant à l'utilisateur d'effectuer plusieurs actions liées à l'analyse et à la manipulation des fichiers Python. Pour commencer, nous avons un large choix de boutons qui s'affiche à l'écran lorsqu'on le lance un bouton pour chaque option possible ces boutons sont : Lister Fichiers Python qui permet de parcourir un répertoire et d'afficher la liste des fichiers Python, Analyser Annotations Type qui permet d'analyses les annotations de type dans un fichier Python spécifié, Ajouter Commentaires qui permet d'ajouter les commentaires pydoc a un fichier Python sélectionné (s'il n'en possède pas), Générer Les Statistiques, qui permet de donner des informations sur un répertoire (comme le pourcentage de fonction avec pydoc, annotation avec typage,…) Ajouter Shebang et UTF-8, qui permet d'ajouter les deux lignes de Shebang et UTF-8 au fichier Python (s'il n'en possède pas), Analyser Deux Premières Lignes qui permet de vérifier la présence des deux premières lignes de commentaire dans un fichier. Analyser Commentaires, qui permet d'analyser les commentaires Pydoc dans un fichier Python. Éditer Fichier, qui permet d'éditer le contenu d'un fichier Python directement dans le GUI. Préférences de Commentaire, qui permet à l'utilisateur de définir son nom d'auteur et le numéro de version pour les ajouts de commentaires Pydoc. Afficher les options possibles, qui permet d'afficher toutes les options de boutons disponibles, Quitter le programme, qui permet de fermer l'application, Choisir Repertoire qui permet de choisir un répertoire, Choisir Fichier qui permet de choisir un fichier. L'Affichage des Résultats de chaque bouton, se fait d'une JTextArea. L'interface est divisée en deux parties, avec une section de boutons à gauche et une zone d'affichage des résultats à droite, simplifiant la lecture. La sélection de fichiers, se fait à l'aide d'un sélecteur de fichiers intégrés pour chaque opération, permettant à l'utilisateur de choisir facilement le fichier ou le répertoire à traiter. Le GUI utilise donc les mêmes méthodes que le mod console, avec des boutons et quelque fonctionnalité supplémentaire comme l'Edition de fichier directement dans l'interface.
